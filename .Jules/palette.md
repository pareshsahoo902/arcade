## 2026-04-18 - Adding ARIA labels to icon buttons\n**Learning:** In plain HTML games relying heavily on canvas and emojis for UI, it is easy to overlook basic accessibility attributes for overlay buttons. Emojis alone don't provide adequate context for screen readers.\n**Action:** Always verify that buttons containing only icons or emojis have descriptive `aria-label` attributes.
## 2026-04-22 - Adding semantic buttons for keyboard accessibility
**Learning:** When programmatically generating interactive UI elements in HTML projects, it is essential to use semantic `<button>` tags rather than `<div>` elements with `onclick` handlers. This automatically provides native keyboard accessibility (Tab focus and Enter/Space activation) and better screen reader support. It's also important to apply CSS resets and define a `:focus-visible` state to mirror `:hover` styles with a clear outline.
**Action:** Always use `<button>` for clickable elements and ensure a clear `:focus-visible` state is defined for keyboard navigation.
